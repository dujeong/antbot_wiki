## Subscribed 토픽

| 토픽 | 타입 | 구독자 | 설명 |
| :--- | :--- | :--- | :--- |
| `/cmd_vel` | `geometry_msgs/Twist` | swerve_controller | 속도 명령 (linear x/y + angular z) |
| `~/direct_joint_commands` | `sensor_msgs/JointState` | swerve_controller | 직접 조인트 명령 <Badge text="타임아웃 5s" variant="caution" /> |
| `/odom` | `nav_msgs/Odometry` | antbot_imu | 정지 상태 감지 (캘리브레이션용) |

### cmd_vel 입력 형식

```
geometry_msgs/Twist:
  linear:
    x: float64   # 전후 속도 (m/s), 양수=전진
    y: float64   # 좌우 속도 (m/s), 양수=좌측
    z: 0.0       # 미사용
  angular:
    x: 0.0       # 미사용
    y: 0.0       # 미사용
    z: float64   # 회전 속도 (rad/s), 양수=반시계
```

<Tabs>
  <TabItem label="전진">
    ```
    # 0.5 m/s로 전진
    ros2 topic pub /cmd_vel geometry_msgs/msg/Twist \
      "{linear: {x: 0.5, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
    ```
  </TabItem>
  <TabItem label="횡이동">
    ```
    # 0.3 m/s로 우측 이동
    ros2 topic pub /cmd_vel geometry_msgs/msg/Twist \
      "{linear: {x: 0.0, y: -0.3, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 0.0}}"
    ```
  </TabItem>
  <TabItem label="제자리 회전">
    ```
    # 1.0 rad/s로 제자리 반시계 회전
    ros2 topic pub /cmd_vel geometry_msgs/msg/Twist \
      "{linear: {x: 0.0, y: 0.0, z: 0.0}, angular: {x: 0.0, y: 0.0, z: 1.0}}"
    ```
  </TabItem>
</Tabs>

## 서비스

<Tabs>
  <TabItem label="적재함 제어">
    **`cargo/command`** <Badge text="antbot_interfaces/CargoCommand" variant="note" />

    | 필드 | 타입 | 값 | 설명 |
    | :--- | :--- | :--- | :--- |
    | Request: `operation` | uint8 | 0 = LOCK | 적재함 잠금 |
    | | | 1 = UNLOCK | 적재함 해제 |
    | Response: `success` | bool | - | 성공 여부 |
    | Response: `message` | string | - | 상태 메시지 |

    ```
    # 적재함 잠금
    ros2 service call /cargo/command antbot_interfaces/srv/CargoCommand "{operation: 0}"

    # 적재함 해제
    ros2 service call /cargo/command antbot_interfaces/srv/CargoCommand "{operation: 1}"
    ```
  </TabItem>
  <TabItem label="전조등">
    **`headlight/operation`** <Badge text="std_srvs/SetBool" variant="note" />

    | 동작 | 명령 |
    | :--- | :--- |
    | <Badge text="ON" variant="success" /> 전조등 켜기 | `data: true` |
    | <Badge text="OFF" variant="danger" /> 전조등 끄기 | `data: false` |

    ```
    # 전조등 ON
    ros2 service call /headlight/operation std_srvs/srv/SetBool "{data: true}"

    # 전조등 OFF
    ros2 service call /headlight/operation std_srvs/srv/SetBool "{data: false}"
    ```
  </TabItem>
  <TabItem label="와이퍼">
    **`wiper/operation`** <Badge text="antbot_interfaces/WiperOperation" variant="note" />

    | 필드 | 타입 | 값 | 설명 |
    | :--- | :--- | :--- | :--- |
    | Request: `mode` | uint8 | <Badge text="0" variant="danger" /> OFF | 와이퍼 정지 |
    | | | <Badge text="1" variant="tip" /> ONCE | 1회 작동 |
    | | | <Badge text="2" variant="success" /> REPEAT | 반복 (주기: cycle_time) |
    | Request: `cycle_time` | float32 | 기본 1.0 | 반복 주기 (초) |

    ```
    # 와이퍼 1회 작동
    ros2 service call /wiper/operation antbot_interfaces/srv/WiperOperation "{mode: 1}"

    # 와이퍼 2초 간격 반복
    ros2 service call /wiper/operation antbot_interfaces/srv/WiperOperation "{mode: 2, cycle_time: 2.0}"
    ```
  </TabItem>
  <TabItem label="컨트롤 테이블">
    **`DirectRead`** / **`DirectWrite`** 서비스를 통해 RCU 컨트롤 테이블 아이템에 직접 접근할 수 있습니다.

    ```
    # 배터리 잔량 읽기
    ros2 service call /direct_read antbot_interfaces/srv/DirectRead \
      "{item_name: 'Battery_Percentage'}"

    # LED 모드 설정
    ros2 service call /direct_write antbot_interfaces/srv/DirectWrite \
      "{item_name: 'Led_Flag_Mode', data: 2}"
    ```

    :::tip
    컨트롤 테이블 전체 아이템 목록은 [5.1 컨트롤 테이블](/antbot_wiki/development-guide/control-table/) 페이지를 참조하세요.
    :::
  </TabItem>
</Tabs>

## 커스텀 메시지

### CargoStatus.msg

적재함 상태를 나타내는 메시지입니다.

<Tabs>
  <TabItem label="메시지 정의">
    ```
    # 도어 상태
    uint8 DOOR_CLOSED = 0
    uint8 DOOR_OPENED = 1
    uint8 door_status

    # 잠금 상태
    uint8 LOCK_NEUTRAL  = 0
    uint8 LOCK_LOCKED   = 1
    uint8 LOCK_UNLOCKED = 2
    uint8 lock_status
    ```
  </TabItem>
  <TabItem label="상태 값 설명">
    **도어 상태**

    | 값 | 상수 | 의미 |
    | :---: | :--- | :--- |
    | <Badge text="0" variant="success" /> | `DOOR_CLOSED` | 도어 닫힘 |
    | <Badge text="1" variant="caution" /> | `DOOR_OPENED` | 도어 열림 |

    **잠금 상태**

    | 값 | 상수 | 의미 |
    | :---: | :--- | :--- |
    | <Badge text="0" variant="note" /> | `LOCK_NEUTRAL` | 중립 |
    | <Badge text="1" variant="success" /> | `LOCK_LOCKED` | 잠금 |
    | <Badge text="2" variant="caution" /> | `LOCK_UNLOCKED` | 해제 |
  </TabItem>
</Tabs>

## QoS 프로파일

각 토픽의 QoS(Quality of Service) 설정:

| 토픽 유형 | Reliability | Durability | Depth |
| :--- | :--- | :--- | :--- |
| `/cmd_vel` | <Badge text="Reliable" variant="success" /> | Volatile | 10 |
| `/odom` | <Badge text="Reliable" variant="success" /> | Volatile | 10 |
| `/tf` | <Badge text="Reliable" variant="success" /> | Volatile | 100 |
| 카메라 이미지 | <Badge text="Best Effort" variant="caution" /> | Volatile | 5 |
| LiDAR 스캔 | <Badge text="Best Effort" variant="caution" /> | Volatile | 5 |
| IMU 데이터 | <Badge text="Best Effort" variant="caution" /> | Volatile | 5 |

:::tip[QoS 매칭 규칙]
퍼블리셔와 서브스크라이버의 QoS 프로파일이 **호환**되어야 통신이 성립합니다.
`Reliable` 퍼블리셔에 `Best Effort` 서브스크라이버는 연결되지만, 반대의 경우는 연결되지 않습니다.
:::
